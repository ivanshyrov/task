# Планировщик задач — система Service Desk

Веб-приложение для регистрации, обработки и контроля заявок ИТ-поддержки с ролевым доступом и синхронизацией данных через SeaTable.

---

## 1. Введение

### 1.1. Назначение системы

**Планировщик задач** — это корпоративная система Service Desk (система управления заявками), разработанная для автоматизации процессов ИТ-поддержки в организации.

Система позволяет:
- сотрудникам создавать заявки на техническую поддержку;
- специалистам ИТ-поддержки выполнять заявки;
- руководителю контролировать процесс выполнения и распределять задачи;
- администратору управлять пользователями, направлениями и настройками системы.

### 1.2. Целевая аудитория

- **Сотрудники организации** — заявики, создающие заявки на техническую поддержку
- **Специалисты ИТ-поддержки** — исполнители заявок
- **Руководители ИТ-отдела** — контроль и распределение задач
- **Системные администраторы** — управление системой

### 1.3. Актуальность

В современных организациях эффективность ИТ-поддержки напрямую влияет на работоспособность сотрудников. Система Service Desk позволяет:
- структурировать поток заявок;
- отслеживать статус выполнения;
- соблюдать SLA (соглашение об уровне сервиса);
- формировать отчётность для руководства.

## 2. Функциональные возможности

### 2.1. Управление заявками

| Функция | Описание |
|---------|----------|
| Создание заявки | Форма с полями: направление, описание, приоритет, срок, вложения |
| Редактирование | Изменение полей заявки (в зависимости от статуса и роли) |
| Удаление | Удаление заявок (по правам) |
| Назначение исполнителя | Назначение ответственного за выполнение |
| Изменение статуса | Переход по статусам: Новая → Назначена → В работе → На проверке → Закрыта |
| Отчёт о выполнении | Текстовое поле с описанием выполненной работы |
| Причина отклонения | Указание причины при отклонении заявки |

### 2.2. Статусы заявок

```
Новая → Назначена → В работе → На проверке → Закрыта
  ↓                              ↓
Отклонена                    (завершено)
```

## 3. Ролевая модель доступа

### 3.1. Роли пользователей

| Роль | Возможности |
|------|------------|
| **admin** | Полный доступ: управление пользователями, направлениями, всеми заявками, настройки, базы данных |
| **director** | Расширенный доступ к заявкам по своей зоне ответственности, просмотр отчётов, базы данных |
| **employee** | Создание заявок, работа только со своими заявками в статусе "Новая" |

### 3.2. Учётные данные по умолчанию

| Логин | Пароль | Роль |
|-------|-------|------|
| admin | admin123 | Администратор |
| director | director123 | Руководитель |
| employee | employee123 | Сотрудник |

## 6. Установка и настройка

### 6.1. Требования

- Node.js 18+
- Аккаунт Vercel
- Аккаунт SeaTable

### 6.2. Развёртывание

1. **Клонирование репозитория**
```bash
git clone <репозиторий>
cd <папка>
```

2. **Установка зависимостей**
```bash
npm install
```

3. **Настройка переменных окружения**

Создать файл `.env.local`:
```env
SEATABLE_API_TOKEN=<ваш токен>
SEATABLE_BASE_UUID=<uuid базы>
SEATABLE_SERVER=https://cloud.seatable.io
SEATABLE_TABLE_NAME=Tasks
SEATABLE_VIEW_NAME=Default View
SEATABLE_USERS_TABLE=Users
SEATABLE_DIRECTIONS_TABLE=Directions
```

4. **Деплой на Vercel**
```bash
npm install -g vercel
vercel deploy --prod
```

### 6.3. Настройка SeaTable

1. Создать базу данных
2. Создать таблицы: Tasks, Users, Directions
3. Настроить колонки согласно таблице 5.x
4. Получить API токен и UUID базы
5. Настроить переменные окружения в Vercel

## 9. Для отчёта (аннотация)

**Планировщик задач** — это система Service Desk для автоматизации процессов технической поддержки, разработанная как веб-приложение с использованием Vanilla JavaScript, HTML и CSS для клиентской части, Node.js serverless-функций для API-слоя и SeaTable в качестве облачной базы данных.
